These endpoints to the development are shown below. The URL to the production server will be provided on request.

  • To access the functional Endpoint, you need to use the bearer token-based authentication.
  • To get the token, The OEM partner number / ID and associated secret key is required to obtain the JWT token, this token will be used for calling various Mastercard Endpoints.

Required parameters for Paywiser Mastercard API Endpoint access
Host (Dev) [suggested to keep in configuration as production level host will be different.] https://dev1onboarding.paywiser.com/BHSKYCAPIV2/BHSKycV2
OEM_Partner_Id / SecretKey Partner ID to be provided Secretkey to be provided
Authorization token type: JWT Receive token to be set at header of the request under authorization. Example: request.AddHeader("Authorization", "Bearer eyJciOi... ");
General notes:

A brief overview of the security setup are provided below:

OEM_Partner_Id this field data will be provided by Paywiser. This is an unique number assigned to each partner.

PWOBUserRefNo this field data will be auto generated by Paywiser. This is an unique number assigned to each applicant. This is the ID that should be used when communicating with Paywiser. It provides Paywiser’s customer service consultant a method to identify the individual in Paywiser’s system.

Customer_Reg_Id this field data should be provided / generated by the partner. This number should be the unique identifier in partner’s internal system. This number should be unique and when Paywiser returns the application status, it would help OEM partner to quickly identify the applicant in their own system.
In case Customer_Reg_Id is not provided, the PWOBUserRefNo will be Customer_Reg_Id.
